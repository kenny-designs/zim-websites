This is a simple problem but before we jump into the code we need to break it down. The same applies to bigger problems. Outlines and diagrams are important to help us plan out what we're doing.

What is the purpose of an outline? The answer is content + summary of the website.

As an example, let's plan out a website for catering. What will this site consist of?

  • Home page
    • Gallery
      • Locations
      • Food
  • Contact information
    • Phone
    • Email
    • Social Media
    • Contact form
  • Services
    • Types of catering they do (weddings, parties, etc.)
    • Delivery
  • Menu
    • Food to buy
Let's say these are our four main tabs that we'd have on a navbar or something. As you can tell, we can break each main category down even further as well.

Let's do another outline but for the CSS of a static site:

  • Home
    • overview
  • Color
    • Color formats (like RGB, Hex, etc)
  • Font
    • Font name
    • Size
  • Images
    • Size
    • Repeat
