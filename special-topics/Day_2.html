Dynamic websites vs Static websites.

Dynamic: Much more dynamic. Data driven. More functionality. LinkedIn, Facebook, etc. Even a simple form. HTML5 Games would be static unless we make it more data driven.
Static: It's static. Not much changes. You get the site and that's it.

Dynamic Site Cons

Client: The web browser. The end user.
Server: The one serving us the website. Database.

The client server model is simple. Say you are calling someone on the phone. The phone is the client, the server is your friend, and if your friend picks up then the server responds. Otherwise, if they don't then it failed. Once again, the phone is the clientand the friend is the server that's getting the request from the phone ie client.

As part of the client-server model, the client sends a request to the server. The server will take a look and see if it can fulfill it. If it can't, it'll 404 or whatever. Otherwise, it'll send back the requested web page or whatever. This response from the server is just that, a response. The client sends a request across the network to a server and then the server sends a response even if it's just a 404 (file not found).

HTML

Hypertext Markup Language. The file extension is .html and .html.
HTML is made up of tags. You know: 

<p>Hello World!</p>

<> is the open tag and </p> is the closing one.
Tags as are essentially content blocks.

The basic structure of HTML is like a sandwhich. It has the head, body, and footer.

Good ol' HTML looks like this: 

<!DOCTYPE html>
<html>
  <head>
    <title>HTML :(</title>
    
    <!-- This is internal CSS. As opposed to inline and external in another file -->
    <style>
      /* We can style tags */
      h2 {
        color: blue;
      }
      
      /* We can style a class */
      .class-style {}
      
      /* We can style identifiers */
      #id-style {}
    </style>
  </head>
  
  <body>
    <!-- The image tag is one of the ones that doesn't need a closing tag -->
    <!-- Also note how the tag can have PROPERTIES (not to be confused with stylings) -->
    <!-- src is an attribute of the image tag -->
  	<img src="./bigSad.png" alt="this text appears if the image cant">
  	
  	<!-- this is the anchor tag. Put in a link and you'll go to whever -->
  	<a href="www.google.com">click here to go to the place</a>
  	
  	<!-- we also have lists -->
  	<!-- ul is unordered lists which uses bullet points -->
  	<ul>
  	  <!-- list item -->
  	  <li></li>
  	</ul>
  	
  	<!-- ol are ordered lists which uses numbers -->
  	<ol></ol>
  	
  	<!-- We also have headers. They go from 1 to 6 -->
  	<h1>BEEG YOSHI</h1>
  	
  	<!-- We can also make a table full of rows and columns -->
  	<table>
  	  <tr>
  	    <td>First column</td>
  	    <td>Second</td>
  	    <td>Third</td>
  	  </tr>
  	</table>
  	
  	<!-- paragraph tag -->
  	<p>This is my paragraph</p>
  	
  	<!-- break tag -->
  	<br>
  	
  	<!-- div tag best tag. Works like a container. You can get crazy with this thing -->
  	<div></div>
  	
  	<!-- Here's an example of inline CSS -->
  	<h1 style="color: red;">Look at my red text!</h1>
  </body>
</html>

By default, HTML is pretty barebones and not very interesting. That's where CSS comes in!

CSS

Cascading style sheets.

There are three types of CSS formatting. External file, internal, and inline. We can have a .css file that we include in the project, we can have the <style> tag and put the CSS code in the HTML file, and inline is where we do it right on the tag itself.

As for today, we're only gonna do internal and inline. External will be next week.

We can style our HTML with CSS via selectors such as .class or #id.
