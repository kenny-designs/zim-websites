We'll be using the same term project from the last two terms. It's just the space invaders game. Class we'll be split into two groups.

There are three basic approaches to building code (imperative, decalarative, and inbetween is object oriented).

For the Imperative approach. Every statement in your code is a command. An instruction to do something. That's why we call the paradigm imperative.

The next form of programming (as seen in scheme) Declarative. It follows the line of thinking of: this what I want, you figure it out.

For a practical example, think of building a house, an imperative appraoch would be get truck, pick up laborers, pick up materials, drive to the house, have laborers get off, take off materials, etc...

The declarative approach to building a house would just go: "Mr. Contractor, here are the blueprints. Make it happen."

Declarative - what is wanted

Between imperative and declarative, we have an inbetween paradigm. It is Object Oriented. Java for example is imperative with OOP extensions. Same goes with C++ since C is imperative. Something like ruby or smalltalk is a real OOP language.

Functional programs are ones in which we can pass around functions like with numerical values.

We'll be using a restricted form of the scheme language. There is exactly one operation. There is no continue, break, goto, etc. Scheme has only one: evaluate which is denoted by (). You evaluate whatever is within the paranthesis. Everything else is done by way of functions. So, (+ 2 2) takes the function + given the parameters 2 and 2. We don't bother with commas as you can probably tell. (functionName arg1 arg2 ...) is the basic pattern. Remember, a single symbol such as + is a function and NOT an operator like how we've come to expect.

In racket, scheme is handled like an interperative language. Source code handled at runtime.

Scheme converts from int to floating point with ease. You can treat numbers like you would treat them on pen and paper when it comes to scheme. We won't be manipulating strings but they are in "" as per usual.

Homework will be in javascript, scheme, etc. As denoted by our professor, this isn't primarily a "coding" class. We'll be doing a bit of everything.
